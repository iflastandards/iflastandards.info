ISBDM string encoding schemes

ISBDM string encoding schemes

This is a set of suggested string encoding schemes for authorized access points for the manifestation and related entities that are applied to the full examples in ISBD for Manifestation.

In this specification, the curly brackets "{}" indicate the element that is the source of a component string. The vertical bar "|" indicates a choice of source, in order of availability. The quotation marks indicate boilerplate strings of punctuation. The plus sign "+" indicates the concatenation of two component strings.

A value of an element sub-type may be used as the source of a component string.

A value should be normalised for a component string, to ensure consistent collocation. Normalisation of a name or title value removes a leading definite or indefinite article.

Manifestation

The base access point for a manifestation is a normalized value of has title proper. If there is no such value, a title supplied by the cataloguing agency may be used.

The string encoding scheme specifies that the base access point is followed by a set of qualifiers enclosed in brackets "(...)" and separated by semicolons "; ".

{normalised(has title proper)}|[supplied title] + "(" {has date of creation of manifestation}|{has date of copyright of manifestation} + "; " + {has creator agent of manifestation} + "; " + {has category of carrier} + ")"

The sequence of qualifiers supports a standard secondary collocation and sort order of authorized access points in a browse index for manifestations.

Other qualifiers are added to the sequence to disambiguate the basic qualified access point.

Added qualifiers are conditional on specific characteristics of the manifestation that are recorded as attribute elements. They support a tertiary collocation and sort order of authorized access points.

Qualify a value of has category of carrier that is "volume" with a value of has binding.

Examples

has authorized access point of manifestation
"Paris apartment (2022; HarperCollinsPublishers; volume; case binding)"
[Full example: Paris apartment (2022; HarperCollinsPublishers; volume; case binding).]
has authorized access point of manifestation
"Evangelistarium Marci Maruli Spalatensis (201-; Nacionalna i sveučilišna knjižnica u Zagrebu; online jedinica građe)"
[Full example: Evangelistarium Marci Maruli Spalatensis (201-; Nacionalna i sveučilišna knjižnica u Zagrebu; online jedinica građe).]

Qualify a value of has category of carrier that is "videodisc" with a controlled term for a finer level of granularity of the category of carrier.

Examples

has authorized access point of manifestation
"Einstein on the beach (2016; Opus Arte; videodisc; DVD)"
[Full example: Einstein on the beach (2016; Opus Arte; videodisc; DVD).]

Qualify a value of has title proper that is a generic title with an authorized access point of an agent who is responsible for the creation of works and expressions that are embodied in the manifestation.

Examples

has authorized access point of manifestation
"Summary records (Unesco. Executive Board. 215th session; 2022; Paris) (2023; Unesco Executive Board; svezak)"

Apply the same string encoding scheme to an authorized access point of a related manifestation.

Examples

has alternate manifestation
"Einstein on the beach (2016; Opus Arte; videodisc; BD)"
[Full example: Einstein on the beach (2016; Opus Arte; videodisc; DVD).]

Item

The base access point for an item is a value of has authorized access point of manifestation.

The string encoding scheme specifies that the base access point is followed by a qualifier that is added to the qualifiers of the access point and separated by a semicolon "; ".

{has authorized access point of manifestation] + insert before ")"("; " + {an identifier or shelfmark of the item})

The qualifier supports a local secondary collocation and sort order of authorized access points of manifestations in a browse index for items.

Examples

has item reproduced by manifestation
"Evangelistarium Marci Maruli Spalatensis (1519; Officina Adamae Petri; svezak; tvrdi uvez; HR-ZaNSK, RIIF-8o-416)"
[Full example: Evangelistarium Marci Maruli Spalatensis (201-; Nacionalna i sveučilišna knjižnica u Zagrebu; online jedinica građe).]

Expression

The base access point for an expression is an authorized access point of the work that is realised by the expression.

The string encoding scheme specifies that the base access point is followed by a qualifier that is added to the qualifiers of the access point, if any, and separated by a semicolon "; ".

{authorized access point of the work] + insert before ")"("; " + {a category of content of the expression} + "; " + {a language of the expression})

The qualifier supports a local secondary collocation and sort order of authorized access points of expressions in a browse index for expressions.

Work

The base access point for a work is a combination of a value of an authorized access point of a creator agent of the work and a value of has title proper of a manifestation that embodies the work.

The string encoding scheme specifies that the base access point is followed by a qualifier that is added to the qualifiers of the access point and separated by a semicolon "; ".

{authorized access point of a creator agent of the work] + ". " + {has title proper}

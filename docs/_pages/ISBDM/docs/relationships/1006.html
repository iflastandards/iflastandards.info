Element reference

Definition
Relates a manifestation to a person.
Scope note
Domain
Manifestation
Range
Person
Element sub-type
has creator person of manifestation has distributor person
Element super-type
has agent associated with manifestation

Additional information

A person may have a form of name that is usually associated with a collective agent. The name may be qualified in an authorized access point to indicate that it is assigned to a person.

Element values

The values of this element may be indexed for uncontrolled or controlled keywords, or used to construct browsing lists.

String encoding schemes

A string encoding scheme should construct the base access point from a name of the person that is the most common in external sources of information.

Apply the guidance for the language or country of origin of the name that is available in Names of persons, if it is considered to be useful for users of the metadata.

For a name of a person that includes a family name, the scheme should transpose the family part of the name to the beginning so that the base appears in a browse index in order of the family name.

A string encoding scheme should use the following strings as qualifiers of a base access point:

  • The date of birth of the person in a standard format
  • The date of death of the person in a standard format
  • A profession, occupation, or field of activity of the person
  • A period of activity that is associated with the profession, occupation, or field of activity of the person

Examples

has person associated with manifestation
"Magritte, René, 1898-1967"
[Full example: Treason of images (1929; Magritte, René, 1898-1967; sheet). The base access point is transposed and is qualified by the dates of birth and death.]

A string encoding scheme should use any strings as qualifiers, if it is considered to be useful for users of the metadata.

Stipulations

Use an appropriate element super-type to record a less specific relationship.

Use an appropriate element sub-type to record a more specific relationship.

Apply the stipulations for the element super-type unless indicated otherwise.

Record a single occurrence of the element that is selected from one or more values that are associated with the manifestation. Use the following order of preference:

  1. Person who is a creator of the manifestation
    1. Person who is a publisher of the manifestation
    2. Person who is a producer of the manifestation
  2. Person who is a manufacturer of the manifestation
  3. Person who is a distributor of the manifestation
  4. Person who is associated with an expression or work that is embodied in the manifestation

Record additional occurrences of the element, if it is considered to be useful for users of the metadata.

Additional information

An identifier of a manifestation is assigned by a cataloguing agency or other organisation by applying a machine-readable enumeration or encoding system.

Consider a stringified IRI for a manifestation to be an identifier of the manifestation. Do not confuse this with an IRI for an identifier string.

Consider an International Standard Serial Number (ISSN) to be an identifier of a diachronic work that is embodied in the manifestation, and not as an identifier of the manifestation.

Record a value that identifies the manifestation. Do not record a value that identifies a sub-unit or super-unit of the manifestation.

Do not record a value that appears in a statement of identifier and terms of availability in the manifestation if it identifies a separate manifestation that is published in a different format or binding.

See also: has manifestation statement of identifier and terms of availability

Element values

The value of an identifier is a literal string (a nomen string).

Record a value that is:

  • a string that is an identifier that appears in an authority file
  • a string that is assigned by an enumeration or encoding system
  • an IRI for the identifier that appears in an authority file, enumeration system, or encoding system

Stipulations

Use the element super-type to record a less specific relationship.

Use an appropriate element sub-type to record a more specific relationship.

Apply the stipulations for the element super-type unless indicated otherwise.

Record a value that appears in an authority file.

Record a single occurrence of the element that is selected from one or more values that are associated with the manifestation. Use any order of preference.

Record additional occurrences of the element, if it is considered to be useful for users of the metadata.

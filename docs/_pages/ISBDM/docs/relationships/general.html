General stipulations for relationship elements

Record a relationship element if it is considered to be useful for users of the metadata, unless indicated otherwise.

Record a value for an entity that is named or otherwise identified in a statement that appears in the manifestation.

Record a value for an entity that is associated with the manifestation in external sources of information.

Record an entity that is probably associated with the manifestation. Record a has note on entity associated with manifestation element that indicates the uncertainty.

Record an entity that is associated with the manifestation. Do not record an entity that is known to be falsely associated with the manifestation.

Record an entity that is real and is known to exist or have existed. Do not record an entity that is known to be fictitious.

Record a has note on entity associated with manifestation element that gives information that is not recorded in a relationship element, if it is considered to be useful for users of the metadata.

If more than one entity is associated with the same relationship of the manifestation, record a separate occurrence of the element for each distinct value that is considered to be useful for users of the metadata. Use the following order of preference:

  1. The prominence of names or titles of entities that appear in statements in the manifestation.
  2. The order of names or titles of entities that appear in a statement in the manifestation.
  3. The predominance of entities that are given in external sources of information.

For a manifestation that embodies the expression of a diachronic work, and if there is a change in the information that is given in statements that describe the manifestation and appear in manifestations that embody expressions of issues or iterations of the work, record an additional occurrence of the element for the entity that has changed, if it is considered to be useful for users of the metadata.

For a manifestation that embodies the expression of a diachronic work, record a has note on entity associated with manifestation element that gives information about a change of entity that is related to the manifestations that embody expressions of issues or iterations of the work, including identification of the issue or iteration in which the change first appears, if it is considered to be useful for users of the metadata.

For a manifestation that embodies the expression of a diachronic work, record a has note on entity associated with manifestation element that gives general information about multiple changes over time of entities that are related to the manifestations that embody expressions of issues or iterations of the work,including identification of the issues or iterations in which the changes occur, if it is considered to be useful for users of the metadata.

See also: has note on entity associated with manifestation

Element values

The value of a relationship element is a reference to a related entity.

Record the value of a relationship element as a literal string or as an IRI (Internationalised Resource Identifier) that is used in semantic linked data.

Record only one kind of value for a single instance of a related entity.

Record different kinds of values for different occurrences of elements, if it is considered to be useful for users of the metadata.

For a linked open data application, use the following order of preference to select the value:

  1. an IRI for the entity
  2. a string that is an identifier of the entity that is a stringified IRI for the entity, derived by enclosing the IRI in string delimiters such as double quotes ("IRI")
  3. a string that is an authorized access point or identifier of the entity that appears in an authority file or similar dataset
  4. a string that is an access point of the entity that is constructed using a string encoding scheme

For a relational database application, use the following order of preference to select the value:

  1. a string that is an authorized access point or identifier of the entity that appears in an authority file or similar dataset
  2. a string that is an access point of the entity that is constructed using a string encoding scheme
String encoding schemes

Use a string encoding scheme to construct an access point of a related entity if an authorized access point is not available.

The string encoding scheme should construct a value that is as unambiguous as possible to support the user task to identify the related entity.

The string encoding scheme should construct a value that consists of a base access point followed by a set of qualifiers. The base access point supports the primary collocation and sort order of access points in a browse index. The qualifiers disambiguate the base access point and are added in a specific sequence to support a secondary collocation and sort order of access points in a browse index.

The string encoding scheme should specify the punctuation marks, labels, or boilerplate strings that are inserted between and within the base access point and the qualifiers to improve clarity and ease of browsing.

ISBDM string encoding schemes is a set of string encoding scheme suggested by ISBD for Manifestation for the value of a related entity.

Sources of information

Use The manifestation or External sources as a source of information for a relationship element.

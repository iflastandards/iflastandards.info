Describing the manifestation

Manifestation that is more than one unit

The units in a multiple unit manifestation are related in a hierarchical structure.

Figure 3 shows that for a multiple unit manifestation there is a choice of which manifestation units to describe as separate related manifestations and which to include in manifestation statements, extent attributes, or notes for the manifestation that is being described. Optional entities, manifestation statements, and notes are outlined by dashes.

For a manifestation that is published or produced in more than one unit, record at least one single description of the unit as a whole, or of one of the sub-units or a super-unit.

For the unit being described, describe the physical characteristics of the unit, sub-units, and super-unit by applying any of the following methods:

  • Record at least one has category of carrier element for the unit and its sub-units. This is mandatory.
  • Record a has extent of unit element or element sub-type for the unit.
  • Record a has extent of unitary structure element for the sub-units.
  • Record a has note on manifestation associated with manifestation element that includes a description of one or more sub-units or a super-unit.
  • Record a has note on manifestation associated with manifestation element that includes a description of one or more sub-units or a super-unit.
  • Record a has manifestation statement element or element sub-type that includes information about the unit, one or more sub-units, or a super-unit.
  • Do not describe the sub-unit or super-unit.

Describe the unit, sub-units, or super-unit with more than one method, if it is considered to be useful for users of the metadata.

Examples

has sub-unit
"Frozen borderline (Nico). Disc one: The marble index"
[Sub-unit of unit described.]
has note on manifestation associated with manifestation
"On two discs with an accompanying booklet."
[Sub-units of unit described.]
has manifestation statement of title and responsibility
"Disc one: The marble index … Disc two: Desertshore …"
[Sub-units of unit described.]
has super-unit
"Frozen borderline (Nico)"
[Super-unit of unit described.]

If a sub-unit or a super-unit has a separate description, record a has sub-unit or a has super-unit element to relate the descriptions.

If no statement appears in the manifestation that distinguishes a sub-unit from other sub-units, describe the sub-unit, if it is considered to be useful for users of the metadata.

Examples

It is not usually useful to describe an individual piece of a jigsaw puzzle.
It is useful to describe an individual piece of a jigsaw puzzle that carries a part of the image that is significant to users of the metadata.
